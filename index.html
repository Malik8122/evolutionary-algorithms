Algorithm Characteristics
Observations

Genetic Programming

Tree-based representation allows symbolic regression — the algorithm discovers both structure and constants. Suitable when interpretable formulas are required. Convergence can stagnate (bloat problem) but mutation and crossover help maintain diversity.

Differential Evolution

Fixed-length real vector representation makes DE highly efficient for continuous optimization. Converges rapidly with well-tuned F and CR. DE/rand/1 strategy with binomial crossover works well across a wide range of problems. Scales predictably with dimensionality.

When to use GP

Symbolic regression, formula discovery, program synthesis, feature engineering, automated ML pipeline construction. Anywhere an interpretable mathematical expression is the desired output.

When to use DE

Continuous parameter optimization, neural network hyperparameter tuning, engineering design, control systems, signal processing, game AI parameter fitting. Problems with a fixed-dimension search space.

EA Variants Overview

Genetic Algorithms (GA)

Fixed-length binary/real chromosomes. Solves combinatorial and continuous optimization. Uses selection, crossover, mutation.

Genetic Programming (GP)

Variable-length tree-based representation for programs/expressions. Evolves structure and constants simultaneously. Used for symbolic regression, program synthesis.

Differential Evolution (DE)

Real-coded population. Mutation via vector differences (donor vector). Binomial crossover. Greedy selection. Excellent for continuous optimization.

Evolution Strategies (ES)

Self-adaptive step sizes. Uses (μ,λ) or (μ+λ) selection. Focuses on continuous optimization with Gaussian mutation. CMA-ES is the modern state-of-the-art variant.

Evolutionary Programming (EP)

Mutations only, no recombination. Self-adaptive mutation strengths. Originally designed for finite state machine evolution. Similar to ES but without recombination.

GP Algorithm Steps
1. Initialize random population of trees (Gen 0)
2. Evaluate fitness of each program
3. Selection via tournament / roulette
4. Apply Crossover (subtree swap)
5. Apply Mutation (random subtree replace)
6. Repeat until termination criterion met
DE Algorithm Steps (DE/rand/1/bin)
1. Init: x_j ~ U[LL, UL] for all j
2. Mutation: v = x_r1 + F·(x_r2 − x_r3)
3. Crossover: u_j = v_j if rand ≤ CR or j == I_rand
               else u_j = x_j
4. Selection: keep min(f(u), f(x))
5. Repeat until convergence
